Git-Page with Hexo and next theme 設定

發表於 2017-08-04

環境背景

我的是 github 是 jason-kuo
所以 hexo-folder-name = jason-kuo.github.io

gitHub 設定

在 gitHub 建立一個 user-name.github.io 的 repository

本機設定

npm install -g hexo-cli     // 全域安裝，只需安裝一次

hexo init hexo-folder-name  // 初始化 hexo-folder
cd hexo-folder-name         // 切換到 hexo-folder    
npm install                 // 安裝 package.json 相依性套件

安裝 Next theme

git clone https://github.com/iissnan/hexo-theme-next themes/next

安裝 deploy 套件

npm install hexo-deployer-git --save

_config.yml 設定

修改下面的區塊設定

deploy:
type: git
repo: https://github.com/jason-kuo/jason-kuo.github.io.git
branch: master

theme: next

hexo\themes\next_config.yml

主題的設定檔

新增 post

將 MD 檔案建立在 source_posts 底下
blog/source/ 下建立 img 資料夾

Hexo 指令

hexo new page “tags” // 建立 tags page
hexo s // 啟動 server

部屬到 github

git init // git 初始化
git add -A
git commint -m “message”

hexo clean // 清除快取檔案( db.json ) 和已產生的靜態檔案( public )
hexo g // 產生頁面到 public folder
hexo d // deploy 到 github

Angular 4 cli 筆記

發表於 2017-08-01

angular cli 指令

ng install -g @angular/cli
**//安裝 angular cli 套件**

* ng new hello
// 建立一個 hello 的目錄，透過 npm 安裝相關的套件

* ng serve
// 啟動 http 伺服器 localhost:4200

* ng g c component-name
// g => generate   c => component
// 建立一個 component-name 的 folder 然後新增相關的檔案
// 註冊到 app.module

* ng g p pipe-name
// p for pipe
// 建立 Pipe 相關的檔案
// 註冊到 app.module

* ng g s service-name
// g => generate s => service

安裝 bootstrap

npm install bootstrap --save

在 style.css 在開頭加上

component

Component 是最小的區塊
Module 是 componet 的 container

一個 App 最少有一個 Component 跟 Module

package.json 的版本數字 ^3.3.7
major.minor.patch
^ 代表可以使用 major 相同的 major 版本的 3.3.8 或 3.3.9 但不能用 4.1 major 版本不同的

Directive (指示)

*ngFor="let c of course"

when using directive to modify structure of DOM adding or remove element , add prefix *

Dependency Injection

app.module 在 @NgModule 的 providers 中註冊要被 DI 的元件

> @Injectable() // 代表可被注入的 Service

Binding

DOM style object

雙向繫結

**Two binding [()] banana in a box 用來記憶語法**

ngModel 是 angular 的雙向繫結語法

**需要在 app.module.ts 引用 FormsModule.**
**import {FormsModule} from '@angular/forms';**
**在 @NgModel 的 imports 屬性陣列中加入 FormsModule**

Build-in pipe

DatePipe at Angular.io

Custom pipe

pipe 跟 component 在 module 中是相同的地位，需要跟 module 註冊

Component API

input property
import { Input } from ‘@angular/core’;

@Input('is-favriote') > decorate a input properties and call it like a function
// is-favriote 是別名(alise - nickName)設定

別名可以讓 template 與 ts 之間的耦合降低，不因為ts 批次改名而出錯. 
keep the contract of your components stable

output event
import { Output,EventEmitter } from ‘@angular/core’;

@Output() change=new EventEmitter( {newValue : this.isSelected} );
在 html 的 event 新增一個 $event object 參數，代表 @Output 傳入的 object 參數

可以將 object 參數的 object 型態寫成 interface 
export 讓外部的 component 引用，形成共用

Shadow DOM

a specification (規範) enables DOM tree and style encapsulation(膠囊化).

Allows us to apply scoped styles to elements
without bleeding out to outer world.

Git-Page with Jekyll 設定

發表於 2017-07-29

環境說明

使用 Jekyll 與 git page 來建立自己的 blog 的備忘錄
操作的本機資料夾根目錄在 c:\github\

git 端設定

在 github 建立 account 帳號
安裝 gitHub Desktop 軟體。

在 gitHub 建立 Repository， 名稱必須為 account-name.github.io
open in desktop 複製到本機資料夾 c:\github\jason-kuo.github.io

安裝 Ruby & bundler

用 gem 指令安裝下面的套件

* gem install jekyll
* gem install bundler

jekyll 安裝指令

把 c:\github\jason-kuo.github.io 中的資料清空
切換到 c:\github 執行下面的指令

jekyll new git-page-name   // 初始化目錄與檔案

cd git-page-name
jekyll serve               // 跑 jekyll server
http://localhost:4000      // 本地端網址

vs code 環境

ctrl + shift + V : 預覽 markdown

新文章設定

在 _posts 中新增 YYYY-MM-DD-name-of-post.markdown 檔案

在開頭至少設定 title
layout: post
title: "設定 Git page"
date: 2017-07-29 16:00:00 +0800
categories:
    -gitpage
    -tageName

_config.yml 中有 index.html 的設定資料，更改之後要 ctrl + C 停止服務 ，重新啟動才會變更
